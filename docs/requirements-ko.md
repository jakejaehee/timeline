# 백엔드 엔지니어링팀 멀티 프로젝트 스케줄링 앱 요구사항

## 1. 제품 목표
이 앱은 백엔드 엔지니어링팀 매니저가 여러 프로젝트를 동시에 운영할 때 다음을 효과적으로 관리할 수 있도록 합니다.

- 프로젝트 진행 상황
- 담당자별 업무 배치 및 실행 순서
- 태스크 간 의존 관계
- 리소스 제약 기반 스케줄링
- 간트차트 시각화
- 경고 및 리스크 탐지

핵심 원칙:
**담당자 리소스 제약을 엄격히 반영한 스케줄링**

---

## 2. 핵심 원칙

### 2.1 관계
- 하나의 프로젝트는 여러 태스크를 가진다
- 담당자는 여러 프로젝트에 참여할 수 있다
- 담당자는 여러 태스크를 가진다
- 태스크는 담당자 기준 실행 순서를 가져야 한다

### 2.2 동시 수행 규칙
- 담당자는 동시에 여러 태스크를 수행할 수 없다
- 단, 소숫점 MD (예: 0.5)인 경우 같은 날 순차 수행 가능

---

## 3. MD 및 Capacity

### 3.1 MD 정의
- 소숫점 1자리까지 허용
- 1.0 MD = 1 working day
- 0.5 MD = 반나절
- 종료 시점은 exclusive

### 3.2 담당자 Capacity
- 1 이하 소숫점 허용
- 실제 소요기간 = MD / capacity

예:
- 1.0 MD / 0.5 = 2일

---

## 4. 비가용일
다음은 모두 비가용일로 처리한다.

- 국가공휴일
- 회사 공통 휴무일
- 개인 휴무

---

## 5. 태스크 순서

### 5.1 개념
- 태스크 순서는 담당자 실행 큐

### 5.2 순서 미지정
- 임시 상태
- 지속적 경고 표시 필요
- 시각적 표시 필요

### 5.3 첫 번째 태스크
- 담당자 기준 첫 태스크는 시작일 필수

### 5.4 UX
- drag & drop 방식
- 담당자 기준 전체 큐 관리

---

## 6. 스케줄 계산

### 6.1 시작일
- 첫 태스크만 필수
- 이후 자동 계산

### 6.2 표시
- 계산된 시작/종료일 표시

### 6.3 시작 계산
actual_start = max(
- 직전 태스크 종료
- 선행 태스크 종료
- 첫 태스크 시작일
- 가용성 제약
)

### 6.4 동일일 규칙
- 정수 종료 → 다음날 시작
- 소숫점 종료 → 동일일 시작 가능

---

## 7. 의존성

- 복수 선행 태스크 허용
- ALL predecessors finished

### Hold / Cancelled
- 의존성 제거 (스케줄 계산에서 제외)
