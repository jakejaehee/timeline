# UI 구조 및 화면 설계 정의

## 1. 전체 UI 설계 원칙

이 앱은 다음 원칙을 기반으로 설계한다.

- 태스크 관리와 순서 편집은 분리한다
- 조회(View)와 편집(Edit)을 분리한다
- 간트차트는 전용 화면에서 크게 제공한다
- 각 화면은 명확한 책임을 가진다

---

## 2. 왼쪽 메뉴 구조

다음 순서를 권장한다.

1. 대시보드
2. 프로젝트
3. 태스크
4. 담당자 스케줄
5. 간트차트
6. 경고 센터
7. 베이스라인
8. 설정

---

## 3. 공통 레이아웃

- 좌측: 사이드 메뉴
- 상단: 글로벌 바
- 중앙: 콘텐츠 영역

상단 바 기능:
- 기간 선택
- 검색
- 빠른 추가
- 경고 배지
- 사용자 설정

---

## 4. 대시보드

### 목적
전체 현황과 리스크를 한눈에 확인

### 구성
- 진행 중 프로젝트 수
- 지연 프로젝트 수
- 순서 미지정 태스크 수
- 일정 충돌 수
- 담당자 workload
- 마감 임박 프로젝트

### 기능
- 프로젝트/경고/담당자 화면으로 이동

---

## 5. 프로젝트

### 리스트 화면
- 프로젝트 목록
- 마감일
- 진행률
- 지연 여부

### 상세 화면
탭 구조:
- 개요
- 태스크
- 일정
- 참여자

### 기능
- 프로젝트 생성/수정
- 태스크 관리
- 프로젝트 간트 확인

---

## 6. 태스크

### 목적
전체 태스크 관리

### 구성
- 테이블 기반 리스트
- 필터 (프로젝트, 담당자, 상태, 중요도 등)

### 기능
- 태스크 생성/수정
- 담당자 지정
- MD 입력
- 의존성 설정
- 상태 변경

---

## 7. 담당자 스케줄

### 목적
담당자별 실행 순서 관리 (핵심 화면)

### 구성
- 좌측: 담당자 리스트
- 중앙: 태스크 큐 (drag & drop)
- 우측: 태스크 상세

### 기능
- 순서 편집
- 시작일 입력 (첫 태스크)
- 순서 미지정 태스크 관리
- 충돌 표시

---

## 8. 간트차트

### 목적
전체 일정 시각화

### 구성
- 좌측 트리 (프로젝트/담당자/태스크)
- 우측 타임라인

### 기능
- dependency 표시
- 마감선
- today 표시
- 필터

---

## 9. 경고 센터

### 목적
문제 및 리스크 관리

### 구성
- 경고 리스트
- 원인 및 해결 가이드

### 기능
- 충돌 확인
- 해결 화면 이동

---

## 10. 베이스라인

### 목적
계획 비교

### 기능
- 스냅샷 저장
- 변경 비교
- 지연 분석

---

## 11. 설정

### 구성
- 공휴일
- 개인 휴무
- 담당자 capacity
- 태스크 종류

---

## 12. 태스크 관리 위치

- 생성/수정: 프로젝트, 태스크 메뉴
- 순서 편집: 담당자 스케줄

---

## 13. 간트 위치

- 메인: 간트차트 메뉴
- 보조: 프로젝트 상세, 담당자 스케줄

---

## 14. 사용자 흐름

### 프로젝트 생성
프로젝트 → 태스크 생성 → 담당자 배정 → 순서 설정 → 간트 확인

### 운영
대시보드 → 경고 확인 → 순서 조정 → 간트 확인

---

## 15. MVP 범위

### 필수
- 대시보드
- 프로젝트
- 태스크
- 담당자 스케줄
- 간트차트
- 경고 센터
- 설정

### 확장
- 베이스라인
- 자동 추천
- 리포트

---

## 16. 핵심 요약

- 태스크 관리와 순서 편집은 분리
- 간트차트는 전용 화면 + 보조 화면 제공
